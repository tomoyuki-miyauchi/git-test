# Step 3: Gitの履歴を探索する

## このステップで学ぶこと

* コミット履歴の確認方法
* Gitがどのように変更履歴を保存しているか
* 過去の状態へ移動する方法
* VS Codeで履歴を確認する方法

---

# なぜ履歴を確認するのか？

Gitの最大の強みは、変更履歴を保存するだけではありません。

履歴を利用することで、

* いつ変更されたのか
* 誰が変更したのか
* なぜ変更されたのか
* 変更前はどうなっていたのか

を確認できます。

ゲーム開発では特に、

> 「昨日まで動いていたのに壊れた」

という状況が頻繁に発生します。

そんなとき、履歴を確認できれば原因調査や復旧が簡単になります。

---

# 現在の履歴を確認する

Step 2では次のようなコミットを作成しました。

```text
Initial commit
Start game documentation
Start developer docs
```

まずは履歴を見てみましょう。

---

# 演習1: コミット履歴を確認する

詳細な履歴を表示します。

```bash
git log
```

コミットごとに

* コミットID
* 作成者
* 日時
* コミットメッセージ

が表示されます。

---

## 一行表示

普段はこちらを使うことが多いです。

```bash
git log --oneline
```

実行例:

```text
762ac02 Start developer docs
16ac970 Start game documentation
9c6ef8a Initial commit
```

---

## グラフ表示

履歴を図として表示できます。

```bash
git log --graph --oneline
```

現在は直線的な履歴なので、あまり面白く見えません。

```text
* 762ac02 Start developer docs
* 16ac970 Start game documentation
* 9c6ef8a Initial commit
```

しかし今後ブランチを利用すると、履歴が枝分かれして表示されるようになります。

---

# コミットIDとは？

各コミットには一意のIDが割り当てられています。

例えば

```text
762ac02
```

や

```text
9c6ef8a
```

のような値です。

このIDを利用すると、過去の任意の時点へ移動できます。

---

# 演習2: 過去のゲームを確認する

Step 2で作成した最初のコミットへ移動してみましょう。

`Initial commit` のコミットIDをコピーします。

例えば

```text
9c6ef8a
```

だった場合、

```bash
git checkout 9c6ef8a
```

を実行します。

---

## 何が起こったか確認する

ファイル一覧を確認してください。

Step 2で作成した

```text
README.md
```

が消えているはずです。

これはGitが

> 「そのコミット時点の状態」

を再現しているためです。

---

## ゲームを確認する

`src/index.html` をプレビューしてみましょう。

ゲーム自体は問題なく動作するはずです。

READMEが存在しない頃のプロジェクト状態へ戻っていることが確認できます。

---

# 最新の状態へ戻る

履歴の確認が終わったら、最新状態へ戻ります。

```bash
git checkout main
```

再度ファイル一覧を確認してください。

```text
README.md
```

が復活しているはずです。

---

# Gitはタイムマシン

ここで重要なのは、

```text
README.md を削除した
```

わけではないということです。

Gitは

```text
過去の状態
↓
現在の状態
```

を切り替えているだけです。

この仕組みによって、

* バグが発生した時点を調査する
* 過去の実装を確認する
* 昔の状態へ戻る

ことができます。

---

# 演習3: VS Codeで履歴を確認する

コマンドラインだけでなく、VS Codeからも履歴を確認できます。

## Graphビューを有効化

1. Source Controlを開く
2. Changesを右クリック
3. Graph を有効化

---

## コミット履歴を確認

Graphパネルに履歴が表示されます。

各コミットを展開すると、

* どのファイルが変更されたか
* どのコミットで追加されたか

を確認できます。

---

## READMEの履歴を追う

次のコミットを探してみましょう。

```text
Start game documentation
```

README.md が追加されたコミットです。

さらに

```text
Start developer docs
```

では README.md が更新されています。

履歴を見ることで、

プロジェクトがどのように成長してきたかを追跡できます。
